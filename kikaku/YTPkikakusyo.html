コンセプト
その人には、その人の
動ける余裕がある。
一般的な予定管理アプリやチェックリストアプリは、予定を並べたり、やることを管理したり、通知で行動を促したりすることを主目的にしている。
しかし本アプリは、それとは異なる。
本アプリの目的は、ユーザーに行動を命令することではなく、予定前に必要な行動をユーザー自身がどの程度重く認識しているかを固定し、その日の状態に応じて「何分前から動くとよいか」を算出することである。
その中心指標として、YTP（余裕適正度ポイント）を用いる。余裕は多ければよいわけではない。少なすぎると焦って崩れ、多すぎるとスマホ・二度寝・別行動に流れる。本アプリは、その人にとっての「適正な余裕」を算出する。
入力を強制しない。
行動を命令しない。
人格を評価しない。
今日の行動認識を固定し、その日のYTPに合わせて予定を組む。

YTPとは
余裕適正度ポイント
YTPは「高い方が良い・低い方が悪い」という指標ではない。その人が、その日に、最も行動開始しやすい余裕幅を数値化したものである。
基礎YTP
初回20問から算出される、その人の基本的な余裕適正。「出発の○分前から動くと安定しやすい」という基準値。
最低YTP
最低限、動ける可能性がある時間。ただし安定ラインではなく、準備漏れや焦りが起きやすい。
適正YTP　※推奨値
最低YTPと上限YTPの中間にある実用上の推奨値。この時間帯が最も安定して行動開始できる。
上限YTP
これ以上余裕を作ると逆に崩れやすい時間。上限を超えた場合、アプリは「もっと早く起きる」ことを勧めない。代わりに行動の組み替えを提示する。
今日のYTP
毎日の状態によって変動するYTP。基礎YTPに眠気・体調・スマホ逸脱リスク・準備の重さ・予定の重要度を加味して算出する。YTPは固定診断ではなく、今日の行動設計値として扱う。
例）最低YTP：20分　／　適正YTP：35分　／　上限YTP：55分
20分前では動けるが不安定。35分前が最も安定しやすい。55分を超えると余裕過多でスマホ・二度寝に流れやすくなる。

HTML / CSS / JavaScript によるブラウザアプリ（実装予定）

外部ライブラリを使わず、HTML / CSS / JavaScript のみで実装する予定。データはlocalStorageに保存し、GitHub Pagesで公開する。スマホ操作前提の設計で、タップ・スクロール操作に最適化する。

データ設計
localStorage による永続化（予定）

基礎YTP・日次YTP・行動Lv別平均時間・セッション履歴をlocalStorageに保存する予定。初回診断結果は基礎YTPとして保持し、日次補正と組み合わせて今日のYTPを算出する。

baseYtp：基礎YTP（初回20問から算出）
actions[]：登録済み行動とLv別平均時間
history[]：過去セッションの出発時刻・開始時刻・実測値
todayYtp：本日のYTP（日次5問で補正）
YTP算出ロジック
2段階の算出式
基礎YTP = 初回20問のスコア合計 → 分に換算
今日のYTP = 基礎YTP + 眠気補正 + 体調補正
　　　　　+ スマホ逸脱補正 + 準備重さ補正 − 予定圧補正
推奨開始時刻 = 出発時刻 − 必要時間
必要時間 = Σ(行動Lv平均) + 切替回数×3分 + バッファ

具体的な流れ
【出発時刻入力 → YTP算出 → 推奨開始時刻】
ユーザーは2つのルートでアプリを使うことができる。行動を登録しなくても使えるのが本アプリの設計思想である。
簡易ルート
出発時刻を入力
→
今日のYTP算出
日次5問で補正
→
推奨開始時刻を表示
例）出発 10:45 ／ 基礎YTP 30分 → 推奨開始 10:15
詳細ルート
出発時刻を入力
→
今日の行動を
Lv分類
→
Lv平均から
必要時間を算出
→
今日のYTPと照合
→
推奨開始時刻を表示
例）Lv1×3、Lv2×1 → 推定必要時間 46分 → 推奨開始 9:59
計算の基本式
必要時間 = Σ(行動Lv平均時間) + 切り替え回数×3分 + 玄関バッファ + 高Lv補正
今日のYTP = 基礎YTP + 眠気補正 + 体調補正 + スマホ逸脱補正 + 準備重さ補正 − 予定圧補正
推奨開始時刻 = 出発時刻 − 必要時間

【アプリ全体の設計方針】
1. 対象ユーザー
予定前の行動開始に困難を感じる人
  • 早めに起きてもスマホや別行動に流れてしまう人
  • 予定直前にならないと動けない人
  • 準備行動の重さが日によって変わる人
  • チェックリストや通知型アプリに圧迫感を覚える人
  • 「余裕を持てば解決する」と言われても、実際には余裕があるほど崩れる人
  • 予定前の準備時間を自分でうまく見積もれない人

※ 本アプリはADHD診断を目的としない。診断の有無に限定せず、予定前行動の見積もりと余裕適正の可視化を目的とする。

2. 開発体制
個人開発（初期版）

HTML / CSS / JavaScript と localStorage を用いたブラウザアプリとして制作し、GitHub Pagesで公開する。MVPの範囲であればひとりで実装可能。将来的に外部公開・検証する場合は、専門家監修やユーザーテストを検討する。

HTML CSS JavaScript localStorage GitHub Pages スマホ操作前提
3. UIの基本方針
命令しない・管理しない・評価しない

アプリはユーザーを動かす命令UIではなく、判断材料を提示するUIにする。チェックリストを主機能にしない理由は3つある。

命令感が出る
「これをやれ・終わったら押せ」という印象になりやすく、本アプリの思想と相反する。
管理されている感が出る
本アプリの目的は行動を完了させることではなく、予定前の行動認識を固定して開始時刻を出すことである。
自己嫌悪につながる
未チェックが残ると「できなかった・また残った」という感覚につながる。完了管理を主目的にしない。

企画の結論
必需品としての
アプリを作る。
01　なぜこの設計思想か
現在、人口の2人に1人がADHD症状を持つと言われている。本アプリはADHD傾向のある人だけを対象にしない。ターゲットを絞らず、なおかつ深く刺さる人をより多くすることが目的である。予定前の行動開始に困難を感じる人は、診断の有無に関わらず広く存在する。その全員に届き得る設計として、YTPという指標にたどり着いた。
02　なぜ続けてもらえるか
黒物アプリ（嗜好品）
ゲーム・YouTube・SNSなど。依存性で継続を促す。なくても何とかなる。
白物アプリ（生活必需品）← YTP
カレンダー・LINE・ブラウザなど。予定がある日には必ず開く。ないと困る。
既存アプリの多くは依存性によって継続を促す。しかし人間はそこまで暇ではない。YTPは予定がある日に必ず開かれる白物として設計されており、習慣ではなく必要性によって使われ続ける。
03　発展構想
将来的には、YTPが同値のユーザー同士がランダムに投稿を見られるプラットフォームへの発展を構想している。同じ余裕適正を持つ人——同じ世界線で生きている人——の生活が見えることで、他者の生活を参照する新しい形が生まれる。既存のSNSとは異なる、行動適正という軸での緩やかなつながりであり、画期的な生活参照の土台になり得ると考えている。
